Description: [HW-2A] 항공사 조회 C15
*&---------------------------------------------------------------------*
*& Report ZHWC15_2A
*&---------------------------------------------------------------------*
*& [HW-2A] 항공사 조회
*&---------------------------------------------------------------------*
REPORT ZHWC15_2A.


*&---------------------------------------------------------------------*
* 선언부
*&---------------------------------------------------------------------*
* 항공사ID를 입력 받는 Parameters를 생성한다.
* 가.Name: PA_CAR
* 나.Type: BC400_S_CARRIER에서 적절한 Component를 선택

PARAMETERS: PA_CAR TYPE BC400_S_CARRIER-CARRID. " BC400_S_CARRIER-CARRID TYPE 의 carrierID 를 입력 받는다.

DATA: GS_CARRIER TYPE BC400_S_CARRIER. " TABLE 조회후 저장을 위한 Structure 를 선언


*&---------------------------------------------------------------------*
* 구현부
* ABAP EVENT에 대한 로직 을 작성
*&---------------------------------------------------------------------*
INITIALIZATION.
*------------------------------------
* Selection Screen 이 샐행되기전 가장 먼저 싱행되는 이벤트
* 주로 Selection Screen 의 기본값 설정 등을 담당
*------------------------------------



AT SELECTION-SCREEN.
*------------------------------------
* Selection Screen 이 출력된 후 사용자의 Enter Or Button Click 마다 작동하는 Event
* Button이 대표적으로 실행버튼(Execute), 이 있다.
*------------------------------------
* 입력된 데이터가 옳바르지 않을경우   START-OF-SELECTION. 구문으로 넘어가지 않고
* 다시 SELECTION-SCREEN 으로 돌려보냄

* 항공사ID를 입력한 후 실행버튼을 누르면, Database Table SCARR를 SELECT문을 사용해서 한 줄만(SINGLE) 조회한다.
* 가. 검색 대상 필드: 전부
* 나. 검색 조건: 입력한 항공사ID와 동일한 데이터
* 다. 검색 결과: 적절하게 선언한 Structure 변수에 SELECT SINGLE로 보관
* 해당 Class Method 확인 (코드첨부)

**&---------------------------------------------------------------------*
** IV_CARRID  TYPE BC400_S_CARRIER-CARRID Airline Code
** ES_RESULT  TYPE BC400_S_CARRIER  Airline Carrier
** NO_DATA    데이터가 존재하이 않습니다.
**&---------------------------------------------------------------------*
*
*  method Z_GET_AIRLINE.
*
** 입력받은 IV_CARRID 가 SCARR TABLE의 CARRID 에 존재할 경우
** SCARR table 에서 ES_RESULT(Strcuture) 로 SINGLE 라인의 정보만 전달
*
*  SELECT SINGLE CARRID CARRNAME CURRCODE URL
*    FROM SCARR
*    INTO CORRESPONDING FIELDS OF ES_RESULT
*    WHERE CARRID = IV_CARRID.
*
** 조회가 안될시 출력하는 내용
*  IF SY-SUBRC <> 0.
*    RAISE NO_DATA.
*  ENDIF.
*
*  endmethod.
*

  CALL METHOD ZCL_2=>Z_GET_AIRLINE
    EXPORTING
      IV_CARRID = PA_CAR               " Airline Code
    IMPORTING
      ES_RESULT = GS_CARRIER              " Airline Carrier
    EXCEPTIONS
      NO_DATA   = 1                " 데이터가 존재하이 않습니다.
      OTHERS    = 2.

* 5.조회 결과가 없는 경우(항공사ID에 대한 정보가 없는) 적절한 오류메시지를 출력한다.
  IF SY-SUBRC <> 0.
    " 검색결과가 없을시 에러 메세지를 출력한다
    MESSAGE E000(Z_2_A) WITH PA_CAR. " PA_CAR가 Target table 에 업을시
                                     " PA_CAR 가 존재하지 않는다는 MESSAGE를 출력
                                     " & 항공사 ID에 대한 정보가 존재하지 않습니다. & 부분에 PA_CAR 가 작성됨

  ENDIF.


START-OF-SELECTION.
*------------------------------------
* AT SELECTION-SCREEN 이 실행된 직후 사용자가 누른 버튼이 실행(Execute)인 경우
* Selection Screen 으로 돌아가지 않고, 이 Event 를 실행한다.
* 이 Event는 Selection Screen 의 입력 값에 따른 데이터 조회를 주 목적으로 하며,
* 조회된 데이터는 특정 로직을 거친 후 출력한다.
*------------------------------------
* 조회 결과 중 MANDT를 제외한 모든 필드를 WRITE문을 사용하여 화면에 출력한다.
  IF GS_CARRIER IS NOT INITIAL. " IS NOT INITIAL 문을 사용하여 GS_CARRIER 변수가 선언 되어 있는지 한번더 확인
    WRITE: / '항공사 ID : ', GS_CARRIER-CARRID,
           / '항공사 이름 : ', GS_CARRIER-CARRNAME,
           / '항공사 코드 : ', GS_CARRIER-CURRCODE,
           / '항공사 URL : ', GS_CARRIER-URL.

 ELSE.
   WRITE: / 'GS_CARRIER Strcutrue가 선언되지 않았습니다. '(001).

  ENDIF.
