*-------------------------------*
  METHOD CONSTRUCTOR.

    IF IV_NUMBER IS INITIAL.
      " 버스 이름이 없음 RAISE
      RAISE EMPTY_NUMBER.

    ELSEIF IV_FARE = 0.
      " 요금이 0 이면 RAISE
      RAISE FARE_ZERO.

    ELSEIF IV_FARE < 0.
      " 요금이 - 값이면 RAISE
      RAISE FARE_MINUS.

    ENDIF.

    " 입력이 성공적이면 입력 값으로 ATTR 을 선언
    MV_NUMBER = IV_NUMBER.

    " 지금 까지 생성된 버스의 수를 +1
    GV_N_O_BUS += 1.

    " OBJ 를 저장하는 ITAB 에 생성된 OBJ 를 APPEND
    APPEND ME TO GT_BUS.

  ENDMETHOD.

    " 승객이 탑승하면  수익에 요금만큼 더한다
    " 이용 승객 수를 +1 한다.
    MV_SALES += MV_FARE.
    MV_N_O_PASSENGER += 1.

  ENDMETHOD.

  METHOD GET_FARE.

    RV_FARE = MV_FARE.

  ENDMETHOD.

    WRITE:/ |버스번호 : { MV_NUMBER } |,
          / |운임     : { MV_FARE } |,
          / |매출액   : { MV_SALES } |,
          / |승객수   : { MV_N_O_PASSENGER } |.

* 모드 버스의 수익을 출력
*-------------------------------*
  METHOD DISPLAY_ALL_SALES.

    DATA: LV_SALES TYPE I. " 전체 수익을 저장하기 위한 LV
    DATA: LO_BUS TYPE REF TO LCL_BUS. " ITAB 에서 OBJ 를 불러올때 사용할 OBJ


    LOOP AT GT_BUS INTO LO_BUS.
      " OBJ 에서 수익을 가져와 모두 더한다.
      LV_SALES += LO_BUS->MV_SALES.

* 이런식으로 가져올 OBJ 를 알면 KEY FIELD - "TABLE_LINE"로  READ TAB  조회가 가능하다
* 근데 의미가 있나? 이미 조호히할 OBJ 를 아는데
*
*    DATA: LO_BUS2 TYPE REF TO LCL_BUS. " ITAB 에서 OBJ 를 불러올때 사용할 OBJ
*      READ TABLE GT_BUS INTO LO_BUS2 WITH KEY TABLE_LINE = LO_BUS.
*      WRITE:/ LO_BUS2->GET_FARE( ).

      ULINE.


    ENDLOOP.

    WRITE :/ |모든 버스의 총 수익의 합은 { LV_SALES } 입니다.|.

  ENDMETHOD.
