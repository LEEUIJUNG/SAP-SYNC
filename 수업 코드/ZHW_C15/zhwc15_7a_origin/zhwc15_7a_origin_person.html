* 승객의 정보를 관리하는 CLASS 구현
*-------------------------------*
CLASS LCL_PERSON IMPLEMENTATION.
  METHOD CONSTRUCTOR.
    " 승객 OBJ 가 생성되면 작동을 함
    IF IV_NAME IS INITIAL.
      " 승객의 이름이 입력 안되면 RAISE
      RAISE EMPTY_NAME.

    ELSEIF IV_MONEY < 0.
      " 승객의 돈이 입력이 안되면 RAISE
      RAISE MONEY_MINUS.
    ENDIF.

    " 승객의 돈과, 이름이 입력되면 승객의 이름과 재산을 입력 값으로 설정
    MV_MONEY = IV_MONEY.
    MV_NAME = IV_NAME.

    " 생성된 승객의 수를 +1 한다.
    GV_N_O_PEOPLE += 1.

  ENDMETHOD.
*-------------------------------*

* 승객이 비용을 내면 작동한다.
*-------------------------------*
  METHOD USE_MONEY.

  IF IV_MONEY < 0.
    " 돈이 없으면 RAISE
    RAISE MONEY_MINUS.

  ELSEIF IV_MONEY > MV_MONEY.
    " 비용 보다 전재산이 적으면 RAISE
    RAISE NOT_ENOUGH_MONEY.

  ENDIF.

  " 비용을 정상적으로 지불
  MV_MONEY -= IV_MONEY.

ENDMETHOD.
*-------------------------------*

* 승객의 전재산을 반환하는 기능
*-------------------------------*
METHOD GET_MONEY.
  " 승객의 전재산을 RETURING 으로 반환한다.
  RV_MONEY = MV_MONEY.

ENDMETHOD.
*-------------------------------*

* 승객의 이름 ,전재산을 출력한다.
*-------------------------------*
METHOD DISPLAY_ATTRIBUTES.

  WRITE:/ |이름   : { MV_NAME }|,
        /  |보유현금 : { MV_MONEY }|.
  ULINE.

ENDMETHOD.
*-------------------------------*

* 지금까지 생성된 승객의 수 를 출력한다.
*-------------------------------*
METHOD DISPLAY_N_O_PEOPLE.

  WRITE:/ |생성된 총 인원 수: { GV_N_O_PEOPLE }|.

ENDMETHOD.

ENDCLASS.
