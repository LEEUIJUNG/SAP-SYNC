REPORT ZHWCC15_1A.

PARAMETERS: PA_NUM TYPE I. " 정수를 입력 받는 파라미터 "

IF PA_NUM < 1.
  MESSAGE ' 입력 받은 정수가 1보다 작습니다. ' TYPE 'E'. " 입력값이 1보다 작을 경우 오류 메시지 출력 "
  "ERROR 메세지는 되도록 사용하지 말고(심각한 ERROR만 사용하자) 오류처럼 보여주고 싶으면 display like를 사용해서 하자"
  " MESSAGE ' 입력 받은 정수가 1보다 작습니다. ' TYPE 'S' DISPLAY LIKE 'I'. 이런식으로 사용 경고처럼 출력하라는 의미

ELSEIF PA_NUM > 100.
  MESSAGE ' 입력 받은 정수가 100보다 큽니다. ' TYPE 'E'. " 입력값이 100보다 클 경우 오류 메시지 출력 "

ENDIF.

* 이런식으로 명확히 작성을 해주면 좋겠다 회사 실무에서 하듯이
*IF PA_NUM < 1.
*  MESSAGE ' 입력 받은 정수가 1보다 작습니다 ( 숫자 범위 1 - 100 ) . ' TYPE 'E'. " 입력값이 1보다 작을 경우 오류 메시지 출력 "
*  WRITE : "출력 내용"(e01). text symbol 사용해서 출력해도 됨
*
*ELSEIF PA_NUM > 100.
*  MESSAGE ' 입력 받은 정수가 100보다 큽니다 ( 숫자 범위 1 - 100 ) . ' TYPE 'E'. " 입력값이 100보다 클 경우 오류 메시지 출력 "
*
*ENDIF.

DATA: GV_SUM TYPE I, " 누적합을 저장하는 변수 "
      GV_INDEX TYPE I. " 반복문을 위한 인덱스 변수 "

* 누적합 계산 및 출력
DO PA_NUM TIMES. " 입력받은 PA_NUM 만큼 반복문 실행 "
  GV_INDEX = SY-INDEX. " GV_INDEX를 loop index 값으로 할당 "
  GV_SUM = GV_SUM + GV_INDEX. " GV_SUM 에 loop index 값 만큼 가산 "

  "ADD GV_INDEX TO GV_SUM 이렇게 작성해도 상관 없음

ENDDO.

WRITE: / ' 입력된 값은 다음과 같다 ', PA_NUM ,' | 누적합 : ', GV_SUM.
