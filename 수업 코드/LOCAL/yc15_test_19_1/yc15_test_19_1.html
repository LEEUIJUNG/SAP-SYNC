REPORT YC15_TEST_19_1.

* 끝에 숫자만 다른 GV_NAME 1 ~ 3 을 선언
DATA: GV_NAME1 TYPE STRING.
DATA: GV_NAME2 TYPE STRING.
DATA: GV_NAME3 TYPE STRING.

DATA: GV_NUMVER TYPE I.

" 이 FIELD-SYMBOL 은 문자열 변수만 가리킬 수 있다.
FIELD-SYMBOLS <FS> TYPE STRING. " <FS> 는 GV_NUMBER 에 추가 할 수 없다 TYPE 이 다르다
" <FS> 는 STRING 변수에 다 접근이 가능하다.

GV_NAME1 = 'GERRARD'.
GV_NAME2 = 'HOMBA'.
GV_NAME3 = 'NORWICH'.

DATA: GV_FIELD TYPE STRING.

* 필드심볼로 어떤 공간에 접근할떄, 아직까지는 그 공간들은 전부 이름을가지고 있는 공간이다. (GV_NAME1 - 3)
* 이름을 가지지 않은 변수에도 접근이 가능하다.(GV_NAME4)
* 동적 프로그래밍이 가능해진다.
DO 4 TIMES. " SY-INDEX 4 ( GV_NAME4 )는 없어 못 가리킨다. 일단 못가리켜도 할당은 가능하다.
  GV_FIELD = |GV_NAME{ SY-INDEX }|.
  ASSIGN (GV_FIELD) TO <FS>.

  IF SY-SUBRC = 0. " ASSGIN이 성공하면 SY-SUBRC 는 0을 반환함.
      WRITE: / <FS>.
  ELSEIF SY-SUBRC = 4. " ASSGIN이 실패하면 SY-SUBRC 는 4를 반환함.
      WRITE: / |{ GV_FIELD }는 존재하지 않는 변수입니다|. " GV_NAME4 는 선언한 적이 없음
      WRITE: / <FS>. " ASSGING 에 실패하면 기존에 가리키던 값을 계속 가리키고 있는다.
  ENDIF.

ENDDO.
