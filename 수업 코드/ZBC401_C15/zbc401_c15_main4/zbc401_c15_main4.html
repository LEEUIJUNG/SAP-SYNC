*&---------------------------------------------------------------------*

FORM CREATE_AIRPLANE USING VALUE(PV_NAME) TYPE STRING
                               VALUE(PV_PLANETYPE) TYPE SAPLANE-PLANETYPE.

* 비행기 OBJ들을 생성
* 생성된 OBJ들을 ITAB에 보관한다.
* OBJ가 생성될때 INSTANCE METHOD - CONSTRUCTOR가 자등으로 호출된다.
  CREATE OBJECT GO_AIRPLANE
    EXPORTING
      IV_NAME      = PV_NAME
      IV_PLANETYPE = PV_PLANETYPE
    EXCEPTIONS
      NO_DATA_FOUND_BY_PLANETYPE = 1
      OTHERS                     = 2
    .

  CASE SY-SUBRC.
    WHEN 0.
      GO_AIRPLANE->DISPLAY_ATTRIBUTES( ).
      ULINE.
    WHEN 1.
      " DB 에 DATA 가 없음
      MESSAGE |해당 비행기 모델로 { PV_PLANETYPE } 검색된 데이터가 없습니다.| TYPE 'E'.
    WHEN OTHERS.
      MESSAGE '알 수 없는 오류가 발생했습니다.' TYPE 'E'.
  ENDCASE.


*  " 굳이 해당 METHOD를 호출할 필요가 없다, OBJ 생성시 CONSTRUCTOR에 의해 자동으로 실행됨
*  GO_AIRPLANE->SET_ATRRTIBUTES(
*  EXPORTING
*    IV_NAME      = PV_NAME
*    IV_PLANETYPE = PV_PLANETYPE
*).

  " 생성된 객채를 OBJ의 주소값을 ITAB에 추가한다.
  APPEND GO_AIRPLANE TO GT_AIRPLANE.

* 이부분의 순서는 상관 없다 어차피 ITAB이 가지고 있는것은 OBJ의 단순 주소값임으로
* APPEND 해서 ITAB에 추가를 한뒤에 작업을 해도 OBJ에 반영이 된다.
*  GO_AIRPLANE->SET_ATRRTIBUTES(
*  EXPORTING
*    IV_NAME      = PV_NAME
*    IV_PLANETYPE = PV_PLANETYPE
*).

ENDFORM.
